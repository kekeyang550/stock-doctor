# 策略回测权益曲线设计

## 背景

策略回测当前已经展示命中样例、胜率、平均收益、最大回撤、收益分布、周期对比和策略对比，但用户只能看到离散交易结果。为了判断样例策略是否“稳”，需要把交易按时间顺序串成一条累计收益和回撤路径。

## 目标

- 在单策略回测响应中新增权益曲线点，展示起点、每笔交易后的累计收益和路径回撤。
- 前端“策略回测”面板新增紧凑路径展示，让用户看到累计收益、最大路径回撤和最近交易节点。
- HTML 研究报告同步导出权益曲线，离线报告能复盘策略路径。
- 不改变现有候选筛选、交易列表排序、周期推荐或策略推荐算法。

## 数据模型

新增 `StrategyBacktestCurvePoint`：

- `step`：路径序号，起点为 0。
- `label`：展示文案，起点为“起点”，交易点为股票名称。
- `equity_pct`：累计净收益百分比，按每笔交易 `return_pct` 累加。
- `drawdown_pct`：相对历史峰值的路径回撤，非正数。
- `trade_return_pct`：该交易自身净收益，起点为 0。
- `symbol` / `name`：交易点对应标的，起点为空。

`StrategyBacktestReport` 新增 `equity_curve: list[StrategyBacktestCurvePoint]`，旧前端缺字段时以空数组兼容。

## 计算规则

- 曲线使用 `exit_date, symbol` 排序后的交易序列，代表交易完成后的路径变化。
- 起点固定为累计收益 0、回撤 0。
- 每笔交易后 `equity_pct += trade.return_pct`。
- `peak_equity_pct` 记录历史最高累计收益。
- `drawdown_pct = equity_pct - peak_equity_pct`，因此最大路径回撤为曲线点中的最小值。
- 数值保留两位小数。

## 前端展示

- 在收益分布后新增“权益曲线”卡片。
- 展示累计收益和最大路径回撤两个摘要指标。
- 展示最多 5 个曲线点，包含交易名称、累计收益、单笔收益、路径回撤。
- 空曲线时隐藏该卡片；旧响应不导致白屏。

## 报告导出

HTML “策略回测”章节新增“权益曲线”小节：

- 展示最后一个曲线点的累计收益。
- 展示曲线最小回撤作为路径最大回撤。
- 列出最多 6 个交易点，包含累计收益、单笔收益、路径回撤。

## 测试策略

- 后端测试先断言 `equity_curve` 存在、起点正确、末点累计收益等于交易收益合计、回撤不为正、交易点带标的信息。
- 前端测试先断言面板显示“权益曲线”“累计收益”“路径最大回撤”和某个交易节点。
- HTML 导出测试先断言导出报告包含权益曲线和路径字段。
- 再实现 schema、服务计算、类型、组件、导出逻辑，最后跑全量后端/前端/构建验证。
