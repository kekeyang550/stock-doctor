# 策略回测收益分布设计

## 背景

策略回测已经展示交易数、胜率、平均收益、最大回撤和收益回撤比。但平均收益容易掩盖样本分布：少数极端收益可能拉高平均值，用户无法快速判断收益是否稳定。

## 目标

给单周期策略回测报告增加收益分布统计：

- 正收益交易数
- 负收益交易数
- 持平交易数
- 收益中位数
- 收益 P25
- 收益 P75

前端策略回测面板和 HTML 报告同步展示这些字段。

## 非目标

- 不改变周期对比推荐算法。
- 不改变策略横向对比算法。
- 不做资金曲线、逐日净值或滚动窗口。
- 不引入外部统计库。

## 后端设计

`StrategyBacktestReport` 新增：

- `positive_trade_count`
- `negative_trade_count`
- `flat_trade_count`
- `return_median_pct`
- `return_p25_pct`
- `return_p75_pct`

分位数使用线性插值：

```text
position = (n - 1) * percentile
value = lower + (upper - lower) * fraction
```

无交易时所有分布字段为 0。

## 前端设计

`StrategyBacktestPanel` 在核心指标之后新增“收益分布”卡片：

- 胜 / 负 / 平
- 中位
- P25 / P75

前端类型将新字段标为可选，旧后端响应缺失时显示 0。

## 报告导出

HTML 报告“策略回测”章节新增“收益分布”，展示同样字段。

JSON 报告继续通过 `strategy_backtest` 原样包含新增字段。

## 测试

- 后端测试断言胜负平数量加总等于交易数，中位数位于最差/最好收益之间，P25 <= P75。
- 前端测试断言策略回测面板展示收益分布。
- HTML 导出测试断言报告包含收益分布。
