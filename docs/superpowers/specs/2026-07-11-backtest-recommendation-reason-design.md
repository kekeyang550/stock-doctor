# 策略回测推荐依据设计

## 背景

策略回测已经支持周期对比、多策略对比和收益回撤比，但当前界面主要用“推荐”“策略推荐”标签表达结果。用户能看到推荐对象，却需要自己从胜率、平均收益、最大回撤和收益回撤比中推断推荐原因。

## 目标

为周期对比和策略对比补充可读的推荐依据，让用户明确知道推荐是如何由当前样例指标得出的。覆盖范围：

- 后端 API 响应
- 前端策略回测面板
- HTML 报告导出
- 前后端测试

## 非目标

- 不改变回测收益、回撤、胜率和收益回撤比的计算口径。
- 不引入新的量化指标。
- 不为每个非推荐项生成长解释，避免界面过重。

## 后端设计

在比较响应中新增字段：

- `StrategyBacktestComparison.recommendation_reason: str | None`
- `StrategyBacktestPresetComparison.recommendation_reason: str | None`

生成规则：

- 没有候选摘要时返回 `None`。
- 有推荐周期时说明推荐持有天数、收益回撤比、平均收益、最大回撤和胜率。
- 有推荐策略时说明推荐策略名称、收益回撤比、平均收益、最大回撤、胜率和交易数量。
- 文案只描述当前样例排序依据，不暗示真实投资收益。

## 前端设计

在“策略回测”面板中：

- 周期对比标题下方显示“推荐依据”短句。
- 策略对比标题下方显示“推荐依据”短句。
- 只有字段存在时展示，接口旧数据或失败态不额外占位。

HTML 报告：

- 策略回测章节在策略横向对比前展示策略推荐依据。
- 若后续报告加入周期对比导出，同一字段可直接复用。

## 测试

后端：

- 服务层断言周期对比和策略对比都返回 `recommendation_reason`。
- API 层断言 JSON 包含字段且文案包含“收益回撤比”。

前端：

- Mock 数据加入两个推荐依据字段。
- App 测试断言策略回测面板展示“推荐依据”与原因内容。
- HTML 导出测试断言报告包含策略推荐依据。
