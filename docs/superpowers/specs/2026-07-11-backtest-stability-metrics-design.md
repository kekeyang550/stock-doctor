# 策略回测稳定性指标设计

## 背景

策略回测已经具备收益分布、收益回撤比、权益曲线和路径回撤。下一步需要回答“这条策略曲线稳不稳”：收益是否剧烈波动、是否连续亏损、路径上是否存在集中盈利或集中亏损。

## 目标

- 在单策略回测报告中新增稳定性指标。
- 前端“策略回测”面板新增“稳定性”卡片。
- HTML 研究报告同步导出这些指标。
- 不改变候选股筛选、交易排序、周期推荐或策略推荐逻辑。

## 指标定义

- `return_volatility_pct`：交易净收益的总体标准差，保留两位小数；交易数为 0 或 1 时返回 0。
- `max_consecutive_loss_count`：按 `exit_date, symbol` 排序后的最长连续亏损交易数量，`return_pct < 0` 计为亏损。
- `best_path_gain_pct`：按时间序列计算的最佳连续收益区间，使用最大连续子序列和，保留两位小数；无交易时为 0。
- `worst_path_loss_pct`：按时间序列计算的最差连续亏损区间，使用最小连续子序列和，保留两位小数；无交易时为 0。

## 展示

前端在“权益曲线”后展示“稳定性”卡片：

- 收益波动：`return_volatility_pct`
- 最长连续亏损：`max_consecutive_loss_count` 笔
- 最佳连续收益：`best_path_gain_pct`
- 最差连续亏损：`worst_path_loss_pct`

旧后端响应缺少字段时默认显示 0，避免白屏。

## 导出

HTML “策略回测”章节在“权益曲线”后新增“稳定性”小节，展示同样四个指标。
