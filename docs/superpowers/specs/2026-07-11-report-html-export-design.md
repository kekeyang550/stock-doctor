# HTML 报告导出雏形设计

## 目标

在 JSON v2 报告包基础上增加一个可直接打开阅读的 HTML 报告导出，让用户不需要解析 JSON 也能查看诊断、组合风险、策略回测和数据可信度摘要。

## 推荐方案

前端新增“导出HTML”按钮，复用同一份 v2 报告 payload，在浏览器端生成自包含 HTML 文件并下载。

选择这个方案的原因：

- 不新增依赖，不引入服务端报告生成。
- 复用 v2 JSON 数据结构，避免 JSON 和 HTML 报告内容漂移。
- 后续可把这份 HTML 作为 PDF 打印或模板化的基础。

## 范围

包含：

- 顶部工具栏新增“导出HTML”按钮。
- HTML 包含诊断摘要、组合风险、模拟仓位、策略回测、数据可信度。
- HTML 内容做基础转义，避免报告字段中的文本破坏结构。
- 导出文件名：`stock-doctor-report-{symbol}-{YYYY-MM-DD}.html`。

不包含：

- PDF/Word 文件生成。
- 图表截图。
- 可编辑模板。
- 服务端持久化报告。

## 测试

- App 测试点击“导出HTML”，捕获 Blob 文本。
- 断言 HTML 包含 `<!doctype html>`、标的名称、组合风险、策略回测和数据可信度。
- 断言文件扩展名为 `.html`。
