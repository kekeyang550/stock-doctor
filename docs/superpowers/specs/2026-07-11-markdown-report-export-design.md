# Markdown 研究报告导出设计

## 背景

当前工作台支持 JSON 报告包和 HTML 可读报告。JSON 适合机器归档，HTML 适合本地打开阅读，但用户如果要把结论粘贴到文档、企业知识库或聊天工具，Markdown 更轻、更容易二次编辑。

## 目标

- 在现有报告 payload 上生成 Markdown，不新增后端接口。
- 内容覆盖诊断摘要、诊断变化/趋势洞察、组合风险、策略回测、数据可信度和复盘行动。
- 文件名与 JSON/HTML 保持一致，只改变扩展名为 `.md`。
- 导出失败沿用现有错误态。

## 内容结构

- 标题：股票名称、代码、导出时间。
- 诊断摘要：评级、综合/技术/资金/风险分、诊断摘要。
- 诊断变化：状态、综合变化、趋势洞察、评级轨迹、风险变化。
- 组合风险：风险等级、权重模式、仓位、行业暴露、风险贡献、再平衡建议。
- 策略回测：价格来源、成本口径、收益/回撤、稳定性、历史对比、最近回测。
- 数据可信度：连接器、刷新、缓存桶与命中率。
- 复盘行动：高/中/低优先级和前若干行动。

## 兼容

- 缺少某个 payload 分支时输出“暂无”或跳过明细。
- Markdown 表格字段会转义竖线，避免破坏表格结构。

## 验收

- 前端测试覆盖点击“导出MD”、文件名、Blob 类型和关键章节内容。
- 前端全量测试和构建通过。
- 浏览器验证按钮可见，页面无白屏和控制台错误。
