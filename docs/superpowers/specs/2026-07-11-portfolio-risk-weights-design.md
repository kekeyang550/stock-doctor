# 组合风险权重模拟设计

## 目标

在“组合风险”面板加入用户输入权重/模拟仓位能力，让用户可以从等权组合切换到更接近真实持仓的风险视角。

## 推荐方案

后端 `/api/v1/risk/portfolio` 增加可选 `weights` 查询参数，格式为 `symbol:weight,symbol:weight`。前端在组合风险面板提供每只自选股的权重输入，用户调整后重新请求组合风险报告。

选择这个方案的原因：

- 默认不传 `weights` 时保持现有等权行为。
- 计算逻辑仍集中在后端，导出和后续扩展可以复用同一份报告。
- 本阶段不需要持久化仓位，避免把模拟输入误认为真实账户。

## 范围

包含：

- 后端解析可选权重，忽略未知代码和非正数。
- 报告新增 `weight_mode`、`total_position_weight`、`positions`。
- 加权平均诊断分、加权平均风险分、加权行业集中度。
- 前端组合风险面板展示权重模式和模拟仓位输入。
- 权重修改后自动刷新组合风险报告。

不包含：

- 持仓成本、数量、市值和盈亏。
- 权重持久化。
- 调仓建议或自动交易。
- VaR、相关矩阵和真实历史波动率。

## 数据模型

新增：

- `PortfolioPositionWeight`

`PortfolioRiskReport` 新增：

- `weight_mode`: `equal | custom`
- `total_position_weight`
- `positions`

`PortfolioRiskDriver` 新增：

- `position_weight_pct`

## 权重规则

- 不传权重时，每只股票等权。
- 传入权重时，仅使用当前组合内且权重大于 0 的代码。
- 如果传入权重无效或总和为 0，回退等权。
- 权重按用户输入总和归一化后参与计算，`total_position_weight` 记录原始有效输入总和。

## 前端行为

- 面板显示“等权模拟”或“自定义权重”。
- 每只自选股显示一个数字输入框，单位为 `%`。
- 用户输入后触发报告刷新；空值或 0 代表该标的在模拟中不参与权重。
- 面板文案明确为“模拟仓位”，不表示真实账户。

## 测试

- 后端服务测试覆盖自定义权重影响平均分、行业集中度和 positions。
- API 测试覆盖 `weights` 查询参数。
- 前端测试覆盖输入权重后重新请求 `/risk/portfolio` 并展示自定义权重模式。
