# 诊断历史趋势洞察设计

## 背景

现有“诊断变化”已经能对比本次和上次诊断，展示综合分、分项分、评级变化和风险变化。但当用户连续保存多份报告后，界面仍偏向“两点对比”，难以快速判断最近几次诊断是在持续增强、持续转弱、评级频繁切换，还是风险分波动扩大。

## 目标

- 在不破坏旧响应结构的前提下，增加最近多次诊断趋势洞察。
- 趋势洞察要覆盖综合分变化、评级变化次数、风险分变化和风险波动范围。
- 前端把多次趋势摘要放在现有“诊断变化”面板内，避免另开复杂页面。
- HTML 研究报告同步导出趋势洞察，方便离线复盘。

## 数据口径

- 后端从已保存的报告历史中筛选同一股票代码，按 `generated_at` 从旧到新取最近若干条，并追加本次实时诊断。
- `score_trend` 继续兼容旧字段，但当有多条历史时扩展为“历史 N / 上次 / 本次”。
- 新增 `trend_insight`：
  - `sample_count`：参与趋势判断的点数。
  - `score_direction`：综合分趋势，取 `up/down/flat/mixed/baseline`。
  - `risk_direction`：风险分趋势，取 `improved/worsened/flat/mixed/baseline`。
  - `rating_change_count`：相邻趋势点评级变化次数。
  - `total_high/total_low/risk_high/risk_low`：最近趋势区间。
  - `summary`：给用户直接可读的中文复盘摘要。

## 交互

- “诊断变化”面板在趋势对比下方展示趋势洞察卡片。
- 卡片突出综合趋势、风险趋势、评级变化次数和区间高低点。
- 旧接口或旧缓存缺少 `trend_insight` 时隐藏该卡片，保持兼容。
- 报告导出新增“趋势洞察”段落，包含摘要和关键指标。

## 验收

- 后端服务测试覆盖无历史、单历史、多历史三类情况。
- API 返回包含 `trend_insight` 且 `score_trend` 能展示多点趋势。
- 前端测试覆盖面板展示和 HTML 导出。
- 全量后端测试、前端测试、前端构建通过。
