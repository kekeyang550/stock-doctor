# 策略回测多策略对比设计

## 背景

当前策略回测已经能查看单个策略的交易样本、周期对比、成本口径和数据血统。用户在研究时仍需要手动切换“强势关注 / 低估值观察 / 资金承压”来比较哪套策略在当前样本下更稳，这个步骤不够直观，也不利于导出报告复盘。

## 目标

新增“多策略横向对比”，在相同持有周期、手续费、滑点和样本数量下，对多套策略生成统一摘要：

- 命中数量
- 交易数量
- 胜率
- 平均收益
- 最大回撤
- 推荐策略
- 数据来源与 fallback 摘要

## 方案

后端在 `StrategyBacktestService` 中新增 `compare_presets()`，复用现有 `run()` 的统计结果，不复制回测逻辑。新增接口：

`GET /api/v1/backtests/strategy/presets`

默认比较 `strong,value,capital-risk` 三个前端主策略，也允许通过 `presets` 查询参数指定其他合法策略。接口参数复用现有回测口径：

- `horizon`
- `holding_days`
- `limit`
- `fee_bps`
- `slippage_bps`

推荐策略规则保持可解释：优先平均收益更高，其次最大回撤更小、胜率更高、交易数更多。收益或交易为 0 的策略仍展示，不隐藏。

## 前端展示

在现有“策略回测”面板内新增“策略对比”区，位置放在“周期对比”之后、交易明细之前。展示三张紧凑策略卡：

- 策略名称
- 命中 / 交易数
- 胜率
- 平均收益
- 最大回撤
- 推荐标签

当前正在查看的策略会高亮，推荐策略会显示“推荐”。对比接口失败时显示“策略对比暂不可用”，不影响单策略回测和周期对比。

## 导出

JSON 报告新增 `strategy_preset_comparison`。HTML 报告在“策略回测”章节内增加“策略横向对比”，展示推荐策略和最多 6 个策略摘要。

## 测试

后端：

- 服务层测试 `compare_presets()` 返回多个策略摘要并给出推荐策略。
- API 测试 `/backtests/strategy/presets` 接受成本、持有周期和样本数量参数。

前端：

- App 测试接口会带当前回测参数请求多策略对比。
- 面板测试多策略对比展示推荐标签和策略名称。
- 导出测试 JSON/HTML 包含多策略对比。
