# 可写操作局部错误恢复设计

日期：2026-07-11

## 背景

前面已经为笔记、价位提醒、系统导入导出补了“进行中”保护，但失败时主要依赖顶部全局错误条。用户需要滚动回顶部才能看到失败原因，而且不知道当前草稿、已有记录或导入预览是否还保留。

## 目标

- 研究笔记保存/删除失败时，在研究笔记面板内显示失败原因，并保留当前草稿或现有笔记。
- 价位提醒保存/删除失败时，在价位提醒面板内显示失败原因，并保留当前输入或现有提醒。
- 系统导出、导入预检、执行导入失败时，在系统存储面板内显示失败原因，并说明已保留当前本地数据或导入预览。

## 交互规则

1. 顶部全局错误条继续保留，作为跨页面兜底。
2. 对应面板新增局部错误块，标题分别为：
   - `研究笔记操作失败`
   - `价位提醒操作失败`
   - `系统存储操作失败`
3. 新一次同类操作开始时清空该面板的旧错误。
4. 保存失败不清空输入草稿。
5. 删除失败不移除原记录。
6. 导入执行失败不清空导入预览，便于用户修复后重试。

## 非目标

- 不新增 toast 系统。
- 不改后端错误格式。
- 不做自动重试。
- 不改变现有顶部全局错误条。

## 验收

- 前端测试覆盖笔记保存失败、价位提醒删除失败、导入预检失败和导入执行失败的局部错误态。
- 前端全量测试、构建和后端测试继续通过。
