# 组合风险再平衡建议设计

## 背景

组合风险面板已经展示模拟仓位、行业暴露和单票风险贡献。用户能看到“谁贡献风险”，但还需要更直接的动作建议：降权、保持、补强，以及建议权重。

## 目标

- 后端组合风险报告新增结构化 `rebalance_actions`。
- 前端组合风险面板展示前 4 条再平衡动作。
- HTML 研究报告同步导出再平衡建议。

## 非目标

- 不做自动交易或真实调仓。
- 不改变现有组合风险分数算法。
- 不引入优化器或外部数学依赖。

## 规则

单票再平衡建议采用启发式：

- 风险分低于 60 或风险贡献较高：建议降权。
- 风险分高于 75 且当前权重不高：建议保持/可补强。
- 其他标的：建议保持观察。

建议权重以当前权重为基础，保守调整：

- 降权：当前权重减 5-10 个百分点，最低不低于 0。
- 补强：当前权重加 5 个百分点，最高不超过 35。
- 保持：建议权重等于当前权重。

## 用户价值

- 用户能从“风险贡献”直接走到“怎么处理仓位”。
- 自定义权重场景下更容易发现单票过重或风险错配。
- 报告导出后保留组合管理建议。
