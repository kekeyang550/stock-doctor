# 策略回测稳定性评估设计

## 背景

策略回测已经展示收益波动、最长连续亏损、最佳连续收益、最差连续亏损等稳定性指标。下一步需要把这些数字转成可读结论，让用户快速判断当前样例策略是“稳定可观察”还是“波动偏高”。

## 目标

- 单策略回测报告新增稳定性评分、等级和解释要点。
- 前端“稳定性”卡片展示评分、等级和解释。
- HTML 研究报告同步导出稳定性评估。
- 不改变候选筛选、交易排序、周期推荐或策略推荐算法。

## 字段

- `stability_score`：0-100 整数，越高表示样例路径越稳定。
- `stability_label`：三档中文标签：
  - `稳定`：评分 >= 75。
  - `需观察`：评分 >= 55 且 < 75。
  - `波动偏高`：评分 < 55。
- `stability_notes`：解释要点列表，说明主要扣分项或优势。

## 评分规则

基础分为 100，按以下维度扣分并限制在 0-100：

- 收益波动：`return_volatility_pct * 4`，最多扣 35 分。
- 连续亏损：`max_consecutive_loss_count * 8`，最多扣 25 分。
- 最差连续亏损：`abs(worst_path_loss_pct) * 2`，最多扣 25 分。
- 平均收益为负时额外扣 15 分。

解释要点：

- 波动 <= 2：提示收益波动较低；否则提示波动需要关注。
- 连续亏损为 0：提示没有连续亏损；否则提示最长连续亏损笔数。
- 最差连续亏损 < -5：提示路径亏损压力偏高；否则提示路径亏损可控。
- 平均收益 >= 0：提示平均收益为正；否则提示平均收益为负。

## 展示

前端在“稳定性”卡片中加入：

- `稳定评分`
- `稳定等级`
- `稳定性说明`，展示最多 4 条说明。

旧响应缺字段时默认评分 0、等级“暂无评估”、说明为空，避免白屏。

## 导出

HTML “稳定性”小节新增稳定评分、稳定等级和说明列表。

## 测试

- 后端测试断言评分范围、等级集合和说明存在。
- 前端测试断言面板显示“稳定评分 / 稳定等级 / 稳定性说明”。
- HTML 导出测试断言离线报告包含同样文案。
