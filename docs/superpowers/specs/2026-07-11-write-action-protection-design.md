# 可写操作交互保护设计

日期：2026-07-11

## 背景

前一阶段已经给刷新任务、复盘行动、热点动作、报告保存、顶部自选和搜索结果加入自选补充了进行中保护。本阶段继续覆盖剩余高频可写操作，避免用户在接口未返回时重复点击，造成重复记录、重复删除或导入导出状态不清晰。

## 范围

- 研究笔记：保存、删除。
- 价位提醒：预设保存、自定义保存、删除。
- 系统存储：导出、导入预检、执行导入。

候选股池目前主要是切换标的，不直接写入服务端；后续如果增加“候选加入自选 / 忽略候选 / 标记跟踪”等动作，再按同一模式补行级保护。

## 交互规则

1. 保存类操作请求未返回前，保存按钮禁用并显示“保存中”。
2. 删除类操作只禁用当前行的删除按钮，并把无障碍标签切换为“删除中...”。
3. 价位提醒保存中，同时禁用预设价位按钮和自定义价位输入，避免同一阶段重复提交。
4. 导出中禁用导出按钮并显示“导出中”。
5. 导入预检中禁用文件选择入口并显示“预检中”。
6. 执行导入中禁用导入按钮并显示“导入中”。
7. 请求失败仍沿用现有全局错误提示；状态必须在 finally 中恢复，避免按钮卡死。

## 非目标

- 不改变后端接口。
- 不增加确认弹窗。
- 不改导入合并策略。
- 不把候选股池变成可写操作。

## 验收

- 前端测试覆盖笔记保存/删除、价位提醒保存/删除、系统导出/预检/导入的 pending 状态。
- 完整前端测试、构建和后端测试继续通过。
