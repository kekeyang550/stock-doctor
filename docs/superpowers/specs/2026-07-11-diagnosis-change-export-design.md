# 诊断变化 HTML 导出设计

## 背景

研究报告 JSON 已经包含 `diagnosis_change`，前端页面也能展示诊断变化、趋势对比、评级轨迹、风险变化和关键驱动。但 HTML 报告只导出诊断摘要，离线阅读时看不到诊断历史对比。

## 目标

在 HTML 研究报告中新增“诊断变化”章节，展示：

- 当前变化状态和评分变化。
- 诊断变化摘要。
- 历史评分趋势。
- 评级轨迹。
- 风险变化。
- 关键驱动。
- 变化明细。

## 非目标

- 不改变 `diagnosis_change` 后端结构。
- 不新增诊断历史算法。
- 不改变页面内诊断变化面板。

## 前端设计

`buildResearchReportHtml()` 从 `payload.diagnosis_change` 读取数据，使用防御式默认值：

- 数组字段缺失时按空数组处理。
- `rating_transition` 和 `risk_shift` 缺失时展示空态文案。
- 所有文本继续通过 `escapeHtml()` 输出。

## 测试

扩展 HTML 导出测试，断言报告包含：

- `诊断变化`
- 诊断变化摘要
- `评级轨迹`
- `风险变化`
- `关键驱动`
- 关键驱动详情
